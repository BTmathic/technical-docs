Introduction

Back when the internet and the World Wide Web were not common household terms, HTML was a standalone language for developing webpages. For example, one could use

<font color='blue'>text</font>

to make an element with its text coloured blue. To add space between two elements we could add an element and colour it the same as the background colour, or add spaces one character at a time with &nbsp;, such    as        this           .

There are two drawbacks to this approach. Firstly, it makes the HTML code a lot more complex if you have a large page with hundreds of styles added throughout the file. More problematic, suppose you have many lines that are all coloured blue and then you need to change the background colour of the page, and the blue text does not look nice anymore. Now you have to go through every single styled HTML tag and change each one (a grep will not help if you are not changing things uniformly either).

This motivated developing a separate language for styling webpages, both cleaner and simpler to use. Many options were considered until 1996 when the World Wide Web Consortium provided a list of recommendations for using Cascading Style Sheets (CSS). Over the years CSS has grown widely to allow more customization and styling options as the internet grew.

The name Cascading Style Sheets comes from the way they are interpreted by the browser, the way the style rules are applied to elements when multiple styles are given, which will be discussed below.

CSS allows you to control how to display HTML elements, how to make them look, or whether to display things or not, all while allowing you to change the display depending on the users device and interaction.

What you should already know

In this doc we assume you are familiar and comfortable with HTML. Only the last section assumes knowledge of Javascript, as we demonstrate a very powerful way to control a document's CSS.

CSS preprocessors

CSS rules are read and set as the page loads, with the HTML and the JavaScript. While very powerful on its own, this limits some styling potential as this means all the data used to render the CSS at the page load is what is used.

The main drawback is that no data can be stored to use as the CSS files are read from top to bottom and the rules are applied via the cascading method. So if you have 100 elements all of the same colour, you have to write the rules out for each one and then if you want to change this colour you need to change them all. However, if you could use a variable, and reference this variable in every rule, you would only need to change the definition of the variable.

To do this, you can use a CSS proprocessor, such as SASS, SCSS, LESS, Stylus or PostCSS. They are similar to CSS but must be compiled into CSS before the browser uses them. This allows simpler code to be written with the same end result, as well as much more functionality such as darkening a colour without needing a new hex code, nesting of style rules to make the page much more readable and much easier to maintain.

While they are highly recommended if you write enough CSS, we will not be discussing them here.

Hello World

To demonstrate the different feautres of CSS discussed here we will be giving rules to style the following simple page

<html>
  <body>
    <p id='id-text'>
      Hello World
    <p>
    <p class='class-text'>
      Goodbye World
    <p>
  </body>
</html>

To style an element we will need to assign the styles to that element specifically. To do this we use selectors to declare what elements a style is to be applied. There are three different ways to access an element in CSS. Firstly, an HTML tag can be accessed by its name, e.g.,

body {
  // styles go here
}

If we tried to do this with the paragraph tag p we would only be able to change both paragraphs at the same time as any style assigned to a paragraph tag styles every paragraph tag. To style individual elements we use their IDs and class names (or pseudo classes below)

#id-text {
  // styles go here
}
.class-text {
  // styles go here
}

Any HTML element with an ID can be assigned styles with #ID, and any element with a class can be assigned styles with .class

Borders

A border has a width, a style and a color, each of which can be assigned using

border-width: // width, e.g., 1px
border-style: // style, e.g., solid
border-color: // color, e.g., black

You can also set a all three border properties in one line with

border: width style color;

The order of the three properties does not matter. We can set

body {
  border: 1px solid black;
}
#id-text {
  border: dashed 2px red
}
.class-text {
  border: double blue 3px
}

The styles available for borders are

dotted, dashed, solid, double, groove, ridge, inset, outset, none, and hidden.

You can also access each side of the border using border-top, border-right, border-bottom, and border-left, and combine these to make more complex borders, e.g.,

border-top: 1px solid black;
border-bottom: 2px dotted blue;

or in one line with

border-color: top-color right-color bottom-color left-color;

or

border-color: top-bottom-color right-left-color;

Note the order here matters, the sides are defined from the top and clockwise.

If a colour is not specified the colour used will be that of the underlying text. The width defaults to 'medium' and the style defaults to 'none'.

Lastly, one can round the edges of a border using

border-radius: size; // e.g., 5px
Code format

Before moving on to the rest of the documentation, it is important to format your code properly in order to get the styles you wish. Compare

body {
  margin: 5px;
  margin-left: 0;
}

to

body {
  margin-left: 0;
  margin: 5px;
}

In the first example we have the margin on the left is 0, while in the second example the margin on the left is 5 pixels. If you have multiple commands styling the same element the webpage is styled with the cascading format: only the last one read by the browser will be used.

To force the browser to ignore this approach, you can give any command precedence over all others by doing the following

body {
  margin-left: 0 !important;
  margin: 5px;
}

With this, the margin on the left is 0 again. If a style is not displaying properly you can try to use this to, but you should generally try to see if re-writing the code will get the styles you want first (e.g., the first example is a better way to remove the margin on the left than using !important). However, if you have imported many different styling packages sometimes and cannot find a simple fix for things being ignored, the important tag can be much simpler than looking through large packages and trying to re-write them.

Width

The width property sets the width of an element in the browser. For example,

p {
  width: size; // e.g., 100px or 50%
}

When using a percentage to determine an elements width, the width of of smallest block containing the element is used.

Height

Similar to width, the height of an element can be set with

p {
  height: size; // e.g., 100px or 50%
}

When using percentage, again, the height of the smallest block containing the element is used.

Margin

There are two main methods to alter the spacing of an element without changing the content inside the element. For example, suppose we want to display our page in the middle of the window, i.e., we want to move this element to the center without changing its size or appearance. The first approach is with margins that give us a way to style how much space should be around an element. Using

margin: 5px;

will put 5 pixels of space around any element it is assigned to. As with borders, we can assign a margin to each side individually using margin-top, margin-right, margin-bottom, and margin-left or in one line as

margin: top right bottom left; // e.g., 1px 2px 3px 5px;

or

margin: top-bottom right-left; // e.g., 5px 10px;

One special value that can be applied to margins is auto. This can allow simple centering of an object in a dynamic environment instead of having to compute how large the element is, how large the page is, and center accordingly. To move the first paragraph horizontally we use

#id-text {
  margin: 0 auto;
  width: 50%;
}

Of course, in this simple case we could simply use

#id-text {
  margin: 0 25% 0 25%;
  width: 50%;
}

instead, but if we are changing the width depending on the device screen size, using auto is simpler.

However, note the following does not change anything with the position of the second paragraph

body {
  height: 200px;
}
.class-text {
  margin: auto 0;
}

This is because the margin-top and margin-bottom are set to 0 for auto, because the height of the containing block is calculated based on its contents, whereas margin-left and margin-right are computed with the width of the containing block, which does not force the width to exactly fit its contents, e.g., a block level element always has 100% width (of its parent block) automatically.

Padding

While a margin adds space around an element, sometimes you may wish to add space inside the element. For example, instead of putting space around the first paragraph, to push the border further out with more space inside we use

padding: 5px;

or to each side individually with padding-top through to padding-left, or the combinations in one line, as above. Compare

p {
  margin: 0;
  padding: 25px;
}

and

p {
  margin: 25px;
  padding: 0;
}
Background

If we don't want specific elements in our page to have a white background, we may use

background: colour // e.g., black, #DDD or #012DEF

The background style will change everything within the border, but not outside, so depending on how you wish to style an element with its background you should select either margin or padding carefully.

Color

If we try to make the background of any of the elements in our Hello World page black, we will not be able to see the text. Similar to the background command we can change the text colour of an element with

color: colour // e.g., white, #333 or #FED210
Links

Something ubiquitous on the web: hyperlinks. While they almost only text on the page, they can be styled much more than plain text because of their functionality.

For example, on an unstyled page a link is displayed in the colour blue, however after you have clicked on the link it will turn purple. There are four different styles one can apply to links in this timeline. To style all hyperlinks on page we use the a selection, and use

  • a:link -- a normal, unvisited link
  • a:visited -- a link after the user has visited
  • a:hover -- a link when the user hovers over it
  • a:active -- a link when it is being clicked

To style particular hyperlinks you simply need to give them an id or class and use the id name or class name instead.

As these occur in a particular order, e.g., the default visited link being purple only occurs after a default blue link has been visited, we must be careful to style these in the correct order.

  • :hover must be used after :link and :visited
  • :active must be used after :hover

Lastly, we can change the default underline as well. To remove the line we can use

a {
  text-decoration: none;
}

Note that in every situation we have a dotted outline when and after clicking a link. This can be removed with

a:focus {
  display: none;
}

However this is not recommended in most situations as the dotted line can be used as an accessibility feature. Namely, for someone that is not using a mouse, the dotted line around the link may be the only way to tell that it is being selected. Removing this would mean some users would not be able to open links without (confusing and frustrating) work, so use this with caution.

Pseudo classes

The styling we saw with links is an example of a pseudo class and not just available for styling links. For example, note that when we hover over a link the mouse changes. This can be controlled with the :hover pseudo-class.

#id-text:hover {
  cursor: pointer;
}

will make hovering the mouse over the first paragraph turn the cursor into a pointer as if you were hovering a link. If you want to make a div a clickable element it may be useful to change the cursor to make this clear to the user.

Other useful pseudo classes include :disabled for when a clickable element is not clickable (e.g., login button without username and password entered), :first-child to style only the first child of an element, :nth-child to style only the n-th child, and :focus (seen above) to distinguish if an element has been selected.

CSS and JavaScript

Pseudo classes allow us to a lot more with styles, pseudo-elements (not covered here) even more, but using JavaScript you can do vastly more.

With JavaScript we can select an element using its HTML tag (selects all elements with this tag) or an ID or class again. For example

let firstParagraph = document.getElementById('id-text');

selects the first paragraph, and to change its style we can use

firstParagraph.style.property = new style;

For example, if we want to change the page to having a grey background and the colour of the text to pink we can use

document.body.style.background = 'grey';
firstParagraph.style.color = 'pink';

In jQuery you can do this with

$(body).css('background', 'grey');
$('id-text').css('color', 'pink');

In more complicated environments like React you can use inline styles or (built in) refs, but in general every JavaScript environment will have ways to access and change the CSS.
