Introduction

Back when the internet and the World Wide Web were not common household terms, HTML was a standalone language for developing webpages. For example, one could use

<font color='blue'></font>
to make text blue. To add space between two elements we could add an element and colour the same as the background or add spaces one character at a time with &nbsp;, such    as        this           .

There are two drawbacks to this approach. Firstly, it makes the HTML code a lot more complex if you have a large page with hundreds of styles added in everywhere. More problematic, suppose you have many lines that are all coloured blue. Then you decide to change the background colour of the page and the blue does not look nice. Now you have to dive through every single one of those styled HTML tags and change each one. This motivated developing a separate language for styling webpages, both cleaner and simpler.

Styling HTML in the before time.

Cascading Style Sheets (CSS), cascading because of how styles are applied (see below)

We can control how to display HTML elements, how to make them look, or whether to display things or not, all while allowing you to change the display depending on the users device and interaction.

Hello World

To demonstrate the different feautres of CSS discussed here we will be styling the following simple page

<html>
  <body>
    <p id='id-text'>
      Hello World
    <p>
    <p class='class-text'>
      Goodbye World
    <p>
  </body>
</html>

To style an element we will need to assign the styles to that element specifically. To do this we use selectors to declare what elements a style is to be applied. There are three different ways to access an element in CSS. Firstly, an HTML tag can be accessed by its name, e.g.,

body {
  // styles go here
}

If we tried to do this with the paragraph tags to style the text elements we would only be able to change both at the same time as any style assigned to a paragraph tag styles every paragraph tag. To style individual elements we use the IDs and class names as follows

#id-text {
  // styles go here
}
.class-text {
  // styles go here
}

Any HTML element with an ID can be assigned styles with #ID, and any element with a class can be assigned styles with .class

Borders

To get a good idea of all the changes we will be making we will give our elements some borders. A border has a width, a style and a color, each of which can be assigned using

border-width: // width, e.g., 1px
border-style: // style, e.g., solid
border-color: // color, e.g., black

You can also set a all three border properties in one line with

border: width style color;

The order of the three properties does not matter. We can set

body {
  border: 1px solid black;
}
#id-text {
  border: dashed 2px red
}
.class-text {
  border: double blue 3px
}

The basic styles available for borders are

dotted, dashed, solid, double, groove, ridge, inset, outset, none, and hidden.

You can also access each side of the border using border-top, border-right, border-bottom, and border-left, and combine these to make more complex borders, e.g.,

border-top: 1px solid black;
border-bottom: 2px dotted blue;

or in one line with

border-color: top-color right-color bottom-color left-color;

or

border-color: top-bottom-color right-left-color;

Do all three properties always need to be present? Example. Rounded edges using border-radius.

Code format

Before moving on to the rest of the documentation, it is important to format your code properly in order to get the styles you wish. Compare

body {
  margin: 5px;
  margin-left: 0;
}

to

body {
  margin-left: 0;
  margin: 5px;
}

In the first example we have the margin on the left is 0, while in the second example the margin on the left is 5 pixels. If you have multiple commands styling the same element, only the last one will be used. While this may seem a little absurb an issue: why would we try to style something twice? This may not be a problem on a small project, but on a larger project, or one where you import other style sheets (e.g., Bootstrap) then it is possible to have elements styled in more place than one. If you are not seeing the CSS you have written being displayed properly, it is possible that the code is correct but different styles are defined elsewhere and take precedence. To deal with this, you can give any command precedence over all others by doing the following

body {
  margin-left: 0 !important;
  margin: 5px;
}

Now we see our margin on the left is 0 again. If a style you want is not displaying properly you can try to use this, but you should generally try to see if re-writing the code will get the styles you want first (e.g., the first example is a better way to remove the margin on the left than using !important). However, if you have imported many different styling packages sometimes and cannot find a simple fix for things being ignored, the important tag can be much simpler than looking through large packages and trying to re-write them.

Margin

There are two main methods to alter the spacing of an element while not changing content inside the element, e.g., suppose we want to display our page in the middle of the window. We want to move this element to the center without changing its size or appearance. The first approach is with margins that give us a way to style how much space should be around an element. For example,

margin: 5px;

will put 5 pixels of space around any element it is assigned to. Again, we can assign each side individually with margin-top margin-right margin-bottom margin-left or in one line as

margin: margin-top margin-right margin-bottom margin-left;

or

margin: margin-top-bottom margin-right-left;

Example ...

One special value that can be applied to margins is auto. This allows us to center an object automatically using CSS instead of having to compute how large the element is, how large the page is, and center accordingly.

body {
  margin: 0 auto;
}

However, note the following does not change anything

body {
  margin: auto 0;
}

Explanation. Width is easy, height is relative; Why?

Background

If we don't want our space, or specific elements in our page, to have a white background, we may use

background: // colour, e.g., black

The background style will change everything within the border, but not outside, so depending on how you wish to style an element with a background you should select either margin or padding carefully. E.g., margin on one, padding on the other.

Color

If we try to make the background of any of the elements in our Hello World page black, we will not be able to see the text. Similar to the background command we can change the text colour of an element with

color: // color, e.g., white
Links

Something ubiquitous on the web: hyperlinks. While they are almost simply text displaying on the page that have a function, there is a lot more to style than simply the colour of the text. For example, on an unstyled page a link is displayed in the colour blue, however after you have clicked on the link it will turn purple. There are four different styles on can apply to links. To style all hyperlinks we have

  • a:link -- a normal, unvisited link
  • a:visited -- a link after the user has visited
  • a:hover -- a link when the user hovers over it
  • a:active -- a link when it is being clicked

To style particular hyperlinks you simply need to give them an id or class and use the id name or class name instead.

You have like seen pages with the default // link blue and visited purple As these occur in a particular order, e.g., visited only occurs after a link has been visited, we must be careful to style these in order. hover after link/visited active after hover What if we don't? Example...

Lastly, we can change the underline as well. // text-decoration Note that in every situation we have the dotted outline when and after clicking a link. This can be removed with focus, however this is not recommended in most situations as the dotted line can be used as an accessibility feature. Namely, for someone that is not using a mouse the dotted line around the link is the only way to tell that it is active. Removing this would mean some users would not be able to open links without much more work. (Imagine a page with links displayed on top of one another, beside on another, in different div elements also styled around the page.)

Cursor

Note that when we hover over a link the mouse changes. This can be controlled with the cursor property. There are many values that can be used, some of the most common are

  • cursor: pointer;
  • cursor: progress;
  • cursor: text;
